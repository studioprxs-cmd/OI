# OI(오늘의 이슈) PRD v1

## 1. 제품 개요
- **제품명:** OI (오늘의 이슈)
- **목표:** 하루 단위 이슈를 빠르게 이해하고, 사용자 참여(투표/베팅/댓글) 기반으로 신뢰도 높은 집단 인사이트를 제공.
- **MVP 범위:** 이슈 생성/조회, 근거 출처 연결, 찬반 투표, 간단 베팅, 댓글, 이슈 결과 확정.

## 2. 문제 정의
- 뉴스/커뮤니티 정보가 파편화되어 핵심 이슈 맥락 파악이 어렵다.
- 사용자는 “무엇이 사실에 가까운가”를 정량적으로 확인하기 어렵다.
- 단순 댓글 기반 토론은 신뢰도/책임성이 낮다.

## 3. 타깃 사용자
- 시사/산업 트렌드에 관심 있는 20~40대
- 빠르게 핵심만 보고 싶은 직장인
- 의견을 남기고 결과를 추적하고 싶은 참여형 사용자

## 4. 핵심 가치 제안
1. **핵심 이슈 요약**: 제목/설명/마감시각 중심의 간결한 카드
2. **근거 기반 토론**: 이슈별 출처 링크(언론/공식자료/데이터)
3. **참여형 예측**: 투표 + 소액 포인트 베팅
4. **사후 검증**: 운영 정책에 따른 결과 확정(Resolution)

## 5. MVP 기능 요구사항
### 5.1 이슈/출처
- 운영자 또는 시스템이 Topic 생성
- Topic별 다중 TopicSource 등록
- 상태: DRAFT / OPEN / LOCKED / RESOLVED / CANCELED

### 5.2 참여
- 로그인 사용자만 Vote/Bet/Comment 가능
- Vote는 사용자당 이슈당 1회(수정 가능 정책은 추후)
- Bet은 잔액 범위 내 포인트 배팅

### 5.3 결과 확정
- 마감 후 운영자 검토로 Resolution 등록
- 확정 결과와 근거 링크를 함께 공개
- 베팅 정산 트랜잭션 기록

### 5.4 신고/모더레이션
- 댓글/이슈/출처 단위 신고(Report)
- 신고 사유 분류 및 상태 추적

## 6. 비기능 요구사항
- 모바일 우선 UI
- API 응답 95p 500ms 이내(캐시 제외)
- 기본 감사 로그(생성/수정/삭제 주체)
- 개인정보 최소 수집(이메일/닉네임)

## 7. 성공 지표(KPI)
- DAU, 이슈 조회수/참여율(투표율, 댓글율)
- 이슈당 평균 출처 수
- 결과 확정 리드타임(마감~확정)
- 신고 처리 SLA 준수율

## 8. MVP 제외 범위
- 실시간 스트리밍 토론
- 고급 평판/등급 시스템
- 외부 결제/실화폐 연동
- 복잡한 추천 알고리즘

## 9. 오픈 이슈
- 베팅 최소/최대 한도 정책
- 투표 변경 허용 여부
- 자동 팩트체크 연동 시점
